# City Traffic Control — 격자 도로망 라이브 관제 (실제 라이브러리 실행)

`cql-native-ai`를 **실제로 실행하는** 도시 교통 관제 데모. 6×6 격자
(36개 교차로 = 36개 도메인, 60개 도로)에 무작위 사고를 주입하면,
정체가 인접 도로를 따라 2차원으로 파도처럼 전파되는 걸 실시간으로
볼 수 있다.

```
브라우저에서 control-room.html 열면 바로 실행됨 — 빌드/서버 불필요
```

## 실제 라이브러리 실행

supply-chain 데모와 동일하게, 재구현이 아니라 esbuild로 컴파일된 실제
`src/` 코드(`cql-native-ai.bundle.js`)를 직접 호출한다:

```
36개 교차로  ->  실제 createAgent()로 만든 DomainAgent 36개
통합 판단     ->  실제 MetaAgent.run() (colimit)
                우측 패널에 "via MetaAgent.run() . contributing=36" 표시
자연성 검사   ->  실제 NaturalTransformation.checkNaturality()
                사고 교차로와 인접 교차로로 자연변환을 구성해 호출
```

## 강남대로 + city 벤치마크의 시각적 완성

이 데모는 앞선 두 작업의 후속이다:

```
강남대로 (3구간 일렬)  ->  1틱 지연 전파를 처음 관찰
city 벤치마크 (2D 격자) ->  그래프거리==전파틱을 수치로 검증 (100%, 편차 0)
이 데모 (6×6 격자)     ->  그 2D 전파를 눈으로 보이게
```

검증 결과, 전파는 완벽하게 규칙적이다: 사고 지점으로부터 **1홉=2틱,
2홉=4틱, 3홉=6틱, 4홉=8틱, 5홉=10틱** — 즉 전파틱 = 그래프거리 × 2
(홉당 정확히 2틱). city 벤치마크는 홉당 1틱이었는데, 이 데모는 회복률을
낮춰(0.035) 사고가 격자 전체(34/35 교차로, 최대 5홉)로 퍼지도록 했기
때문에 홉당 2틱이 된 것이며, 거리-틱 선형 관계 자체는 동일하게 성립한다.

## 전파 메커니즘에 대한 정직한 한계

인접 전파는 강남대로/supply-chain과 동일한 "이전 틱 참조" 패턴을 쓴다
— 라이브러리 정식 History 기능이 아니라 시뮬레이터 내부 구현이며,
논문 Limitations (4)에 명시된 것과 같은 한계다. 각 교차로의 상태 판정,
전체 통합(MetaAgent.run), 자연성 검사(checkNaturality)는 실제 라이브러리
호출이다.

## 조작

```
▶ 자동 실행   매 틱 진행 + 22% 확률로 무작위 사고
⚡ 사고 발생   무작위 교차로에 사고 주입 → 격자로 전파 관찰
▶| 한 틱       한 틱씩 수동 진행
↺ 초기화       전체 리셋
```
